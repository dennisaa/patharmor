DR_EXPORT void dr_init ( client_id_t  client_id)

When registering a process, users must provide a list of paths to client libraries and their associated client-specific options. DR looks up "dr_init" in each client library and calls that function when the process starts. Clients can register to receive callbacks for the various events within dr_init. Note that client paths and options cannot include semicolons. client_id is the ID supplied at registration and is used to identify the client for dr_get_options(), dr_get_client_path(), and external nudges.
